## ファイル配布方法

### 方法1：QRコードでURL共有（推奨）

1. `index.html` を GitHub Pages または Netlify にアップロードする
2. 公開されたURLを QR コードに変換する
3. 参加者に QR コードを読み込んでもらう

#### GitHub Pages での公開手順

```bash
# リポジトリ作成後、以下を実行
git init
git add index.html
git commit -m "Add workshop app"
git branch -M main
git remote add origin https://github.com/ユーザー名/リポジトリ名.git
git push -u origin main
```

その後、GitHub の Settings → Pages → Source: main / root で公開。
URL: `https://ユーザー名.github.io/リポジトリ名/`

#### Netlify Drop での公開（最速・無料）

1. https://app.netlify.com/drop にアクセス
2. `index.html` ファイルをドラッグ＆ドロップ
3. 自動でURLが発行される（例: `https://random-name.netlify.app/`）

### 方法2：QRコード生成ツール

公開URLが決まったら、以下のサービスでQRコードを生成：

- **QR Code Generator**: https://www.qr-code-generator.com/
- **Googleフォームのリンク**: Googleクラスルームで配布も可
- **大きいQRコード**: プロジェクター投影用に400×400px以上で作成

### 方法3：ローカル配布（Wi-Fiなし環境）

```bash
# Python 3 がある場合、簡易HTTPサーバーを起動
python3 -m http.server 8080

# 同一Wi-Fiネットワーク内のスマホから
# http://[PCのIPアドレス]:8080/ にアクセス
```

## ワークショップ進行ガイド

### タイムライン

| 時間 | 内容 | ファシリテーターの役割 |
|------|------|----------------------|
| 0〜5分 | 導入・QRコード配布 | アプリURLをQRで共有、キャラクター紹介 |
| 5〜25分 | **第1幕**（発災直後の初動） | 島ごとに選択肢を話し合わせる |
| 25〜30分 | 島内振り返り① | 「なぜその選択をしたか」を共有 |
| 30〜50分 | **第2幕**（通信・情報収集） | 自分のクリニックではどうか問いかける |
| 50〜55分 | 島内振り返り② | EMISを知っているか確認 |
| 55〜70分 | **第3幕**（患者・クリニック対応） | 要配慮者の話し合いができているか確認 |
| 70〜75分 | ハザードマップ確認 | 各自が自分の地域を確認 |
| 75〜80分 | まとめ・アクションプランニング | アクションを書いてスクリーンショット保存 |

### 各幕の進め方

**基本ルール**
1. まずグループで「どうするか」を話し合う（1〜2分）
2. 全員が選択肢を選んでから次へ進む
3. 解説画面を読み合わせる
4. 「振り返り時間」にファシリテーターが問いかけをする

**振り返りの問いかけ例**

第1幕後：
- 「実際に発災したとき、最初に何をしようと思いましたか？」
- 「家族への連絡手段は何を使っていますか？」

第2幕後：
- 「あなたのクリニックのスタッフへの連絡手段はいくつありますか？」
- 「EMISを知っていますか？登録していますか？」

第3幕後：
- 「要配慮者の患者さんと、避難先の話をしたことがありますか？」
- 「クリニックの棚は固定されていますか？」

### キャラクター
- **ネコミヤ ニャンタ**：ねこやま県またたび市 マタタビクリニック院長

| 幕 | タイトル | 学習目標 |

- **通信**：複数手段・EMIS登録・患者への連絡手段
- **お金**：運転資金・休業補償保険・BCP文書

## トラブルシューティング

### アプリが開かない
- ブラウザを最新版に更新してください（Safari / Chrome 推奨）
- プライベートモードを解除してみてください

### 途中で落ちてしまった
- 「もう一度プレイする」ボタンで最初から再開できます
- 途中保存機能はありません（各幕20〜25分で完結するため）

### スマホ以外でも使える？
- PC・タブレットでも動作しますが、スマホ縦持ち最適化です
- PC で見ると画面が左に寄ることがあります（仕様）
